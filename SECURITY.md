# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in TrackFlow, please report it privately rather than opening a public issue.

**Contact:** itmesamtat@gmail.com  
**Subject line:** `[SECURITY] <brief description>`

Please include:
- A description of the vulnerability
- Steps to reproduce it
- The potential impact
- Any suggested fixes if you have them

I aim to respond within 48 hours and will work to release a fix as quickly as possible. I'll credit you in the release notes if you'd like.

## Scope

The following are in scope:
- The TrackFlow desktop application (Windows)
- The TrackFlow backend (Supabase database, Edge Functions, RLS policies)
- The TrackFlow website (gnznaki.github.io/TrackFlow-Site)

## Out of Scope

- Vulnerabilities in third-party dependencies (report these upstream)
- Issues that require physical access to the user's machine
- Social engineering attacks

## Supported Versions

Only the latest release is actively supported with security updates.

| Version | Supported |
|---|---|
| Latest | Yes |
| Older | No — please update |
